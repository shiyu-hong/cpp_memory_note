2  对象

2.1 基础概念

在C++中，对象是具有生命周期且占用存储空间的实体。从创建到销毁的整个过程中，对象始终占据内存中的特定区域。这片内存空间不仅用于存储对象的成员变量，还承载着对象的状态信息。每个对象都隶属于一个特定类型，该类型决定了对象所包含的成员变量和成员函数。

2.2 对象类型分类

C++中的对象类型可分为多个类别，它们在内存管理和生命周期方面各具特点。主要的对象类型包括算术类型、指针类型、数组类型、枚举类型、类类型和结构体类型等。而非对象类型则涵盖函数类型、引用类型和void类型等。深入理解这些分类，有助于开发者更高效地进行内存管理并编写出高质量的代码。

以下是这些对象类型的简要说明及特点：

对象类型 简要说明 内存管理特点
算术类型 包括整型、浮点型等基本数据类型 由编译器自动分配和释放内存
指针类型 存储其他对象的地址 需要手动管理内存，容易产生内存泄漏
数组类型 一组相同类型的数据的集合 在栈或堆上分配内存，需要注意数组越界问题
枚举类型 由一组命名的常量组成的类型 由编译器自动分配和释放内存
类类型 包含数据成员和成员函数的自定义类型 通过构造函数和析构函数管理内存
结构体类型 与类类似，但成员默认为公有 与类类型相似

以下是非对象类型的简要说明及特点：

非对象类型 简要说明 特点
函数类型 表示可执行代码的类型 通过调用执行，在栈上分配
引用类型 为已存在对象提供别名 不占用独立内存，随原对象管理
void类型 表示无类型，通常用于函数返回值或指针 无独立实例，用于类型转换和泛型编程

通过对这些类型特点的理解，开发者可以更高效地进行内存管理，并编写出高质量的代码。

2.3 对象类型判定

C++标准库提供了编译期类型特征工具，可用于判断类型是否为对象类型。

2.3.1 标准库类型特征

  • C++11：引入类模板std::is_object，通过模板特化机制在编译期完成类型判断。

  • C++17：引入变量模板std::is_object_v，作为std::is_object<T>::value的语法糖。

2.3.2 代码示例

#include <iostream>
#include <string>
#include <type_traits>

/**
 * @brief 检查类型是否为对象类型并打印结果
 *
 * @tparam T 待检测的类型
 * @param type_name 类型名称
 */
template <typename T>
void print_is_object_type(const std::string& type_name) {
  constexpr bool is_object{std::is_object_v<T>};

  std::cout << std::boolalpha;
  std::cout << "Type '" << type_name << "' is an object type: " << is_object << '\n';
}

int main() {
  // 测试算术类型
  print_is_object_type<int>("int");
  // 测试指针类型
  print_is_object_type<int*>("int*");
  // 测试引用类型
  print_is_object_type<int&>("int&");
  // 测试函数类型
  print_is_object_type<decltype(main)>("decltype(main)");
}

2.3.3 运行结果

Type 'int' is an object type: true
Type 'int*' is an object type: true
Type 'int&' is an object type: false
Type 'decltype(main)' is an object type: false

2.4 生命周期

对象的生命周期始于构造函数的完成，终于析构函数的开始执行。在此期间，对象占据已分配的内存，并可通过其成员函数和变量进行操作。在C++中，对象的存储期决定了它们的创建和销毁时机，具体情况如下：

2.4.1 自动存储期

  • 典型示例：代码块内的局部变量。

  • 生命周期：对象在控制流进入作用域时被创建，离开作用域时销毁。例如，在函数中定义的局部变量，当函数被调用时创建，当函数执行完毕时销毁。

  • 内存管理：在运行时，系统自动处理内存的分配与释放。

2.4.2 静态存储期

  • 典型示例：全局变量、命名空间作用域变量、静态局部变量、静态数据成员。

  • 生命周期：在程序启动时创建，在程序终止时销毁。

  • 特点：在程序整个运行期间都存在；未显式初始化的变量会被自动置为零。

2.4.3 动态存储期

  • 典型示例：使用new表达式创建的对象以及由智能指针管理的对象。

  • 生命周期：对象通过显式分配创建，在以下情况被销毁：

    • 调用delete或delete[]时。

    • 智能指针超出作用域或满足释放条件时。。

  • 内存管理：需要手动控制内存的分配和释放。建议采用智能指针来实现自动管理，同时遵循RAII原则，以防止内存泄漏。

2.4.4 线程存储期

  • 典型示例：使用thread_local关键字声明的变量。

  • 生命周期：对象在线程启动时创建，在线程结束时销毁。

  • 内存管理：为每个线程创建独立副本，有效避免多线程环境下的数据竞争问题。

2.4.5 临时对象

  • 典型示例：函数返回的临时值、类型转换时创建的临时对象，以及字面量。

  • 生命周期：

    • 临时对象通常仅在当前完整表达式内有效，表达式结束时即被销毁。

    • 当临时对象绑定到常量左值引用或右值引用时，其生命周期会延长至引用结束时。

  • 注意事项

    • 临时对象无法绑定到非常量左值引用。

    • 自C++11起，临时对象可以绑定到右值引用T&&，从而支持移动语义。

    • 仅当临时对象直接绑定到局部引用变量时，其生命周期才能延长；若函数返回内部临时对象的引用，则会导致悬空引用。

  • 代码示例

    #include <iostream>
#include <string>
#include <vector>

/**
* @brief 演示常量左值引用可延长临时对象生命周期
*
* @note 临时对象存活至引用作用域结束
*/
void test_const_lvalue_reference() {
  const auto& ref = std::string("temp"); // 临时对象绑定到常量引用
  std::cout << "const ref: " << ref << '\n';
  // ref销毁时，临时对象一并销毁
}

/**
* @brief 演示临时对象不能绑定到非常量左值引用
*
* @note 编译错误：右值无法绑定到非常量左值引用
*/
void test_nonconst_lvalue_reference() {
  auto& ref = std::string("error"); // 编译错误
}

/**
* @brief 演示右值引用绑定临时对象及移动语义
*
* @note 右值引用可修改临时对象，支持移动操作
*/
void test_rvalue_reference() {
  auto&& rref = std::string("Hello,"); // 临时对象绑定到右值引用
  rref += "World!";                    // 可修改临时对象

  std::vector<std::string> vec;
  vec.push_back(std::move(rref)); // 移动语义：资源转移，避免拷贝

  std::cout << "rref (moved): " << rref << '\n'; // 移动后rref为空
  std::cout << "vec[0]: " << vec[0] << '\n';     // 打印：Hello,World!
}

/**
* @brief 错误示例：返回局部临时对象的引用
*
* @return 悬空引用（函数结束后对象已销毁）
* @warning 禁止返回局部对象的引用
*/
const std::string& broken_heart() {
  return std::string("error"); // 危险：临时对象在函数结束时销毁
}

/**
* @brief 演示悬空引用的未定义行为
*
* @warning 函数返回的临时对象无法通过引用延长生命周期
*/
void test_lifetime_rule() {
  const auto& leaked = broken_heart();                     // leaked指向已销毁的内存
  std::cout << "broken heart: " << broken_heart() << '\n'; // 未定义行为：导致程序崩溃
}

int main() {
  test_const_lvalue_reference();    // 常量左值引用 - 生命周期延长
  test_nonconst_lvalue_reference(); // 非常量左值引用 - 编译错误
  test_rvalue_reference();          // 右值引用 - 移动语义
  test_lifetime_rule();             // 悬空引用 - 程序运行时崩溃
}
