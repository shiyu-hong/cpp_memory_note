1  字节

1.1 概述

在C++程序运行期间，内存用于存储代码指令和数据。从抽象层面来看，内存由若干离散分布的连续内存块组成，每个内存块均可视为一段连续排列的字节序列。

1.1.1 字节特性

在C++程序中，字节不仅是数据的基本存储单元，也是内存的最小可寻址单元。每个字节对应唯一的内存地址，程序可通过地址访问任意字节。

1.1.2 字节位数

字节由若干连续的二进制位组成，关于其位数，需注意以下几点：

  • 最小位数：每个字节至少包含8个二进制位。在绝大多数现代计算平台上，一个字节恰好为8位。

  • 实现定义：字节的具体位数由实现定义。在某些嵌入式系统或特殊架构的平台上，字节位数可能为16位甚至32位。

1.2 工具箱

1.2.1 查询字节位数

C++标准规定 char 类型的大小始终为1字节。基于这一特性，可通过引入标准库头文件 <climits>, 调用其中的预定义宏 CHAR_BIT，来查询当前运行平台下1字节包含的二进制位数。

以下是一个简单的代码示例：

#include <climits>
#include <iostream>

int main() {
  // 输出当前平台的字节位数
  std::cout << "Number of bits in a byte on this platform: " << CHAR_BIT << '\n';
}

运行结果如下：

Number of bits in a byte on this platform: 8

1.2.2 获取对象占用的字节数

在C++中，sizeof 是编译期运算符，用于获取对象或类型在内存中占用的字节数，其返回值的类型为无符号整数类型 size_t。

以下是一个简单的代码示例：

#include <iostream>

int main() {
  // 使用 sizeof 运算符获取 int 类型占用的字节数
  size_t int_bytes{sizeof(int)};

  // 定义浮点变量 flag，初始值设为 3.14
  double pi{3.14};
  // 获取 pi 变量占用的字节数，sizeof 作用于变量时括号可省略
  size_t double_bytes{sizeof pi};

  // 依次输出 int 和 double 类型占用的字节数
  std::cout << "Size of int: " << int_bytes << '\n';
  std::cout << "Size of double: " << double_bytes << '\n';
}

运行结果如下：

Size of int: 4
Size of double: 8

1.3 注意事项

1.3.1 布尔类型的内存占用

从逻辑上看，C++中的布尔类型仅需1个二进制位即可存储 true 和 false 两种状态，但由于字节是内存的最小可寻址单元，布尔类型在内存中仍会占用1字节的存储空间。

以下是一个简单的代码示例：

#include <iostream>

int main() {
  // 定义布尔变量 flag，初始值设为 false
  bool flag{false};

  // 使用 sizeof 获取 flag 的内存大小，并输出结果
  std::cout << "Size of boolean variable: " << sizeof(flag) << " byte.\n";
}

运行结果如下：

Size of boolean variable: 1 byte.
